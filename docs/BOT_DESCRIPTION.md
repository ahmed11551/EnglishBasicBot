# Описание программного комплекса «МГИМО BASIC ENGLISH»

## 1. Общее описание системы

Программный комплекс «МГИМО BASIC ENGLISH» представляет собой интегрированную образовательную платформу, предназначенную для изучения базовой англоязычной лексики (семья, еда, путешествия, повседневная жизнь). Система ориентирована на широкий круг пользователей начального и среднего уровня владения языком (A1–B1).

Архитектурно система состоит из двух взаимосвязанных компонентов:

1. **Telegram Mini App (Progressive Web Application)** — клиентское веб-приложение, реализующее полнофункциональную среду обучения с множеством режимов тренировки, алгоритмом интервального повторения, системой прогресса и геймификации. Приложение запускается как внутри мессенджера Telegram в качестве Mini App, так и автономно в любом современном браузере.

2. **Telegram-бот** — серверный чат-бот, функционирующий в среде мессенджера Telegram и предоставляющий пользователю возможность получать ежедневные порции лексики, проходить викторины и осваивать новые термины непосредственно в диалоговом интерфейсе мессенджера без необходимости перехода в отдельное приложение.

Предметная область охватывает более 400 английских и 350 испанских терминов, систематизированных по тематическим модулям, отражающим ключевые направления международных отношений: дипломатия и протокол, система ООН и международные организации, безопасность и кризисное реагирование, основы международных отношений, государственное устройство, экономика и торговля, международное право, общество и культура, академическая лексика, а также устойчивые фразы и коммуникативные формулы.

Каждая лексическая единица включает: термин на иностранном языке, перевод на русский язык, транскрипцию в формате IPA (для английского языка), пример употребления в контексте, а также опциональную ссылку на аудиофайл произношения.

## 2. Архитектура приложения

### 2.1. Клиентская архитектура (Mini App)

Веб-приложение реализовано в парадигме одностраничного приложения (Single Page Application, SPA) с клиентской маршрутизацией. Архитектура следует принципу разделения ответственности и включает следующие уровни:

- **Уровень представления** — React-компоненты, организованные по страницам (`pages/`) и переиспользуемым элементам интерфейса (`components/`). Каждый экран приложения представлен отдельным компонентом: `Home`, `Topics`, `ModuleDetail`, `FlashCards`, `Trainer`, `Builder`, `Listen`, `Pronounce`, `Search`, `Stats`, `Settings`.

- **Уровень управления состоянием** — централизованное состояние приложения реализовано посредством React Context API (`AppContext`). Контекст инкапсулирует данные о прогрессе пользователя, статистике по модулям, дневном журнале активности, настройках (дневная цель, язык интерфейса, звук) и идентификаторе Telegram. Все мутации состояния осуществляются через мемоизированные функции обратного вызова (`useCallback`), что обеспечивает оптимальную производительность при перерисовках.

- **Уровень бизнес-логики** — реализация алгоритма интервального повторения SM-2 (`spacedRepetition.ts`), утилиты для работы с аудио и синтезом речи (`utils/audio.ts`, `utils/sounds.ts`), модуль распознавания речи (`utils/speechRecognition.ts`).

- **Уровень персистентности данных** — модуль `storage.ts` обеспечивает сериализацию и десериализацию состояния в `localStorage` браузера. Ключи хранения параметризуются идентификатором пользователя Telegram, что позволяет изолировать данные различных пользователей на одном устройстве.

- **Уровень данных** — статические массивы лексических карточек (`data/cards.ts`, `data/spanishCards.ts`) и описания модулей (`data/modules.ts`), типизированные с помощью TypeScript-интерфейсов.

Маршрутизация реализована с применением библиотеки React Router. Для оптимизации начальной загрузки используется ленивая загрузка компонентов (`React.lazy`) с разделением кода на уровне маршрутов (code splitting). Компоненты, не участвующие в первичном отображении (тренажёр, конструктор, аудирование, произношение, статистика, настройки), загружаются по требованию, что существенно сокращает объём начального JavaScript-бандла.

Обработка ошибок реализована посредством компонента `ErrorBoundary`, который перехватывает исключения в дереве React-компонентов и отображает информативное сообщение об ошибке вместо полного краха интерфейса.

### 2.2. Серверная архитектура (Telegram-бот)

Серверная часть реализована в виде бессерверной функции (Serverless Function), развёрнутой на платформе Vercel. Функция-обработчик вебхука (`api/webhook.js`) принимает HTTP POST-запросы от серверов Telegram и обрабатывает входящие сообщения и нажатия на inline-кнопки.

Архитектура бота включает:

- **Обработчик вебхука** — основная точка входа, осуществляющая маршрутизацию входящих событий (текстовые команды, callback-запросы от inline-кнопок) к соответствующим обработчикам.

- **Модуль данных** (`api/wordsData.js`) — серверная копия лексической базы, предоставляющая функции получения «слов дня», случайных слов и формирования вопросов для викторины.

- **Хранилище подписок** — база данных подписчиков на ежедневную рассылку, реализованная на основе Vercel KV (управляемое хранилище Redis). Идентификаторы чатов подписчиков хранятся в множестве (Redis Set) под ключом `daily_words_subscribers`.

- **Планировщик рассылки** — задание по расписанию (Vercel Cron), выполняющееся ежедневно в 12:00 по московскому времени и отправляющее порцию из пяти терминов всем подписчикам.

## 4. Функциональные возможности веб-приложения (Mini App)

### 4.1. Главный экран «Сегодня»

Главный экран приложения реализует концепцию ежедневного обучения и выполняет функцию информационной панели (dashboard). Экран содержит следующие элементы:

- **Кольцо дневной цели** — круговой индикатор прогресса, отображающий соотношение изученных за текущий день слов к установленной дневной цели (от 5 до 30 слов, настраивается пользователем). Визуальная анимация кольца обеспечивает наглядную обратную связь о прогрессе.

- **Счётчик серии дней (streak)** — отображает количество последовательных дней, в которые пользователь выполнял хотя бы одно упражнение. Серия рассчитывается на основе дневного журнала активности (`dailyLog`).

- **Календарь недели** — визуальное представление активности за последние семь дней в формате цветовой матрицы, где интенсивность цвета отражает количество изученных слов в соответствующий день.

- **Блок «Слова на сегодня»** — список карточек, запланированных алгоритмом интервального повторения SM-2 для повторения в текущий день. Формируется на основе параметра `nextReviewAt` каждой карточки.

- **Блок «Недавно изученные»** — отображает карточки, с которыми пользователь взаимодействовал за последние 7 дней, отсортированные по дате последнего просмотра.

- **Блок «Часто ошибаешься»** — агрегация карточек с наибольшим значением счётчика ошибок (`wrongCount`), с возможностью быстрого перехода к повторению проблемных слов.

- **Быстрый доступ к модулям** — плитки тематических модулей с индикацией прогресса и возможностью продолжить обучение с последнего открытого модуля.

### 4.2. Тематические модули

Лексический материал организован в тематические модули:

**Английский язык (10 модулей):**
1. Diplomacy & Protocol (Дипломатия и протокол)
2. UN & Organizations (ООН и организации)
3. Security & Crisis (Безопасность и кризисы)
4. Core Concepts (Основы международных отношений)
5. State & Governance (Государство и власть)
6. Economy & Trade (Экономика и торговля)
7. International Law (Международное право)
8. Society & Culture (Общество и культура)
9. Academic (Академическая лексика)
10. Phrases & Communication (Фразы и общение)

**Испанский язык (8 модулей):** аналогичная тематическая структура, адаптированная для испаноязычной лексики международных отношений.

Каждый модуль отображается с индикатором прогресса (progress bar), показывающим долю изученных карточек от общего числа в модуле. Типизация модуля включает: уникальный идентификатор, название на английском и русском языках, текстовое описание, массив идентификаторов карточек, цвет обложки и эмодзи-иконку.

### 4.3. Режимы обучения

Приложение предоставляет пять различных режимов тренировки, каждый из которых задействует различные когнитивные механизмы запоминания:

**Флэш-карты (Flash Cards)** — классический режим карточек с возможностью пролистывания (swipe). Лицевая сторона отображает термин на иностранном языке и транскрипцию (IPA). Кнопка воспроизведения озвучивает термин через Web Speech API (SpeechSynthesis) или предзаписанный аудиофайл. При переворачивании карточки отображается перевод на русский язык и пример употребления в контексте.

**Тренажёр (Trainer)** — режим множественного выбора. Пользователю предъявляется термин на иностранном языке и четыре варианта перевода, один из которых является правильным. Ответ сопровождается визуальной и звуковой обратной связью. Результат ответа передаётся алгоритму SM-2 для корректировки расписания повторения.

**Конструктор (Builder)** — режим составления слова из перемешанных букв. Пользователю предъявляется перевод термина, и необходимо последовательно выбрать буквы, составляющие исходный термин на иностранном языке. Режим тренирует навык активного воспроизведения орфографии.

**Аудирование (Listen)** — режим восприятия на слух. Пользователю воспроизводится произношение термина без отображения текста, после чего предлагается выбрать правильный перевод из четырёх вариантов. Режим развивает навык аудиального восприятия профессиональной лексики.

**Произношение (Pronounce)** — режим тренировки произношения с использованием микрофона. Пользователю отображается термин, который необходимо произнести вслух. Распознавание речи осуществляется через Web Speech API (SpeechRecognition). Результат распознавания сопоставляется с эталонным произношением, и пользователю предоставляется обратная связь о качестве произношения.

### 4.4. Поиск

Модуль глобального поиска позволяет осуществлять полнотекстовый поиск по всей лексической базе — как по терминам на иностранном языке, так и по их переводам на русский язык. Поиск выполняется на стороне клиента с фильтрацией в реальном времени по мере ввода запроса.

### 4.5. Статистика

Экран статистики предоставляет пользователю агрегированную информацию о процессе обучения:

- **Система опыта (XP)** — каждое правильное выполнение упражнения начисляет очки опыта. Накопление XP определяет уровень пользователя.
- **Серия дней (streak)** — непрерывная последовательность дней с активным обучением.
- **Прогресс по модулям** — визуальные индикаторы (progress bars) для каждого тематического модуля.
- **Слова на повторение** — количество карточек, подлежащих повторению согласно расписанию SM-2.

### 4.6. Настройки

Экран настроек предоставляет следующие параметры персонализации:

- **Дневная цель** — количество слов для ежедневного изучения (от 5 до 30), регулируемое ползунком.
- **Звук** — включение/отключение звуковых эффектов.
- **Язык интерфейса** — выбор из трёх вариантов: Русский, English, Español. Переключение языка автоматически применяет соответствующую цветовую тему.
- **Сброс прогресса** — возможность полного обнуления данных обучения.

## 5. Функциональные возможности Telegram-бота

Telegram-бот реализует следующий набор команд и интерактивных элементов:

### 5.1. Команды

- **/start** — приветственное сообщение с описанием возможностей бота и inline-клавиатурой для быстрого доступа к функциям: «Слова дня», «Учить», «Квиз», «Приложение», «Подписка на рассылку».

- **/words** — отправка пяти терминов дня с форматированием: жирный термин, перевод, пример употребления (курсив), перевод примера (скрыт под tg-spoiler для самопроверки). Кнопка «Ещё 5 слов» позволяет получить дополнительную порцию.

- **/learn** — пошаговый режим изучения. Бот отправляет один термин, скрывая перевод. Кнопка «Показать перевод» раскрывает значение и пример через редактирование сообщения (`editMessageText`). Кнопка «Следующее» загружает новый термин.

- **/quiz** — викторина с множественным выбором. Бот отправляет термин и четыре кнопки с вариантами перевода. При нажатии на кнопку отображается результат (верно/неверно) с правильным ответом и кнопка перехода к следующему вопросу.

- **/subscribe** — подписка на ежедневную рассылку слов дня. Идентификатор чата пользователя сохраняется в Redis-множестве (Vercel KV).

- **/unsubscribe** — отмена подписки на рассылку. Идентификатор чата удаляется из Redis-множества.

- **/app** — отправка ссылки для открытия Mini App непосредственно в интерфейсе Telegram.

### 5.2. Inline-кнопки

Все взаимодействия с ботом реализованы через inline-клавиатуру (InlineKeyboardMarkup), что обеспечивает интуитивный интерфейс без необходимости ввода текстовых команд. Обработка нажатий осуществляется через механизм callback-запросов Telegram Bot API.

### 5.3. Ежедневная рассылка (Cron)

Подсистема рассылки функционирует на основе планировщика Vercel Cron. Ежедневно в 12:00 по московскому времени серверная функция извлекает список подписчиков из Vercel KV, формирует порцию из пяти терминов и отправляет её каждому подписчику. Механизм обработки ошибок обеспечивает продолжение рассылки при недоступности отдельных чатов.

## 6. Алгоритм интервального повторения SM-2

Система запоминания лексики основана на алгоритме SuperMemo 2 (SM-2), разработанном П. Возняком. Алгоритм оптимизирует расписание повторений для каждой карточки на основе качества предыдущих ответов пользователя.

### 6.1. Параметры модели

Для каждой карточки хранятся следующие параметры:

- **easeFactor** (фактор лёгкости) — вещественный коэффициент, инициализированный значением 2.5 и находящийся в диапазоне [1.3; 2.5]. Отражает субъективную сложность карточки для конкретного пользователя.
- **interval** (интервал) — текущий интервал повторения в днях.
- **repetitions** (количество повторений) — число последовательных успешных повторений.
- **quality** (оценка качества) — числовая оценка от 0 до 5, где 0–2 соответствуют ошибочному ответу, 3 — «трудно», 4 — «хорошо», 5 — «легко».
- **nextReviewAt** — временная метка (timestamp) следующего планового повторения.
- **wrongCount** — кумулятивный счётчик ошибочных ответов для формирования блока «часто ошибаешься».

### 6.2. Логика пересчёта

При качестве ответа ниже порогового значения (quality < 3) карточка считается неосвоенной: счётчик повторений обнуляется, интервал устанавливается в ноль, повторение назначается через 1 минуту, а фактор лёгкости уменьшается на 0.2 (но не ниже 1.3).

При успешном ответе (quality ≥ 3) интервал рассчитывается по следующим правилам:

- При первом успешном повторении: интервал = 1 день.
- При втором: интервал = 6 дней.
- При последующих: интервал = предыдущий интервал × easeFactor.

Фактор лёгкости пересчитывается по формуле SM-2:

```
EF' = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))
```

где `q` — оценка качества ответа, `EF` — текущий фактор лёгкости. Результат ограничивается диапазоном [1.3; 2.5].

### 6.3. Выборка карточек для повторения

Функция `getDueCardIds` определяет карточки, подлежащие повторению в текущий день. Карточка включается в выборку, если: (а) для неё отсутствует запись о прогрессе (новая карточка), (б) значение `nextReviewAt` не превышает конец текущего дня, либо (в) количество повторений равно нулю.

## 7. Система прогресса и геймификации

Для повышения мотивации и вовлечённости пользователя в приложении реализованы следующие механизмы геймификации:

- **Дневная цель** — настраиваемая цель по количеству слов (5–30), визуализированная в виде кругового индикатора на главном экране. Достижение цели сопровождается специальным звуковым сигналом.

- **Серия дней (streak)** — отслеживание непрерывной последовательности дней обучения. Серия рассчитывается ретроспективно по дневному журналу: начиная с текущего дня, проверяется наличие активности за каждый предшествующий день, пока не встретится день без активности.

- **Система опыта (XP) и уровни** — каждое выполненное упражнение начисляет очки опыта. Накопленный XP определяет текущий уровень пользователя, отображаемый на экране статистики.

- **Визуальная и звуковая обратная связь** — при правильном ответе воспроизводится положительный звуковой сигнал, при ошибочном — отрицательный. Звуковые эффекты генерируются программно через Web Audio API (осциллятор с заданной частотой и длительностью) и могут быть отключены в настройках.

- **Прогресс по модулям** — для каждого модуля отслеживается количество изученных карточек (карточки с `repetitions > 0`), число слов на повторение и дата последней тренировки.

## 8. Многоязычная поддержка и тематическое оформление

### 8.1. Языки интерфейса

Приложение поддерживает три языка интерфейса: русский (по умолчанию), английский и испанский. Выбор языка осуществляется в настройках и сохраняется в `localStorage`.

### 8.2. Цветовые темы

Система тематического оформления реализована через CSS-переменные (CSS Custom Properties) с двумя предустановленными темами:

- **USA** — основная тема для русского и английского языков. Цветовая палитра: тёмно-синий, белый, красный — цвета, ассоциируемые с англоязычным миром.

- **España** — тема для испанского языка. Цветовая палитра: красный и жёлтый — цвета флага Испании.

Переключение темы происходит автоматически при смене языка интерфейса. Технически переключение реализовано через атрибут `data-theme` корневого HTML-элемента, который активирует соответствующий набор CSS-переменных.

## 9. Офлайн-режим и PWA

Приложение реализовано как Progressive Web Application (PWA) и обеспечивает полноценную работу в отсутствие сетевого подключения.

### 9.1. Service Worker

Генерация Service Worker осуществляется средствами плагина `vite-plugin-pwa` на этапе сборки проекта. Service Worker перехватывает сетевые запросы и обеспечивает кеширование статических ресурсов (HTML, CSS, JavaScript, шрифты, изображения) по стратегии «Cache First» с предварительным кешированием (precaching) при первой загрузке.

### 9.2. Манифест веб-приложения

Файл `manifest.webmanifest` описывает метаданные приложения для операционной системы: название, описание, иконки различных разрешений (192×192, 512×512), цветовую схему, режим отображения (`standalone`) и начальный URL. Это позволяет пользователю установить приложение на домашний экран мобильного устройства и запускать его в полноэкранном режиме без элементов интерфейса браузера.

### 9.3. Хранение данных

Вся пользовательская информация (прогресс, статистика, настройки, дневной журнал) хранится в `localStorage` браузера. Благодаря этому данные доступны в офлайн-режиме без обращения к серверу. Ключи хранения параметризуются идентификатором Telegram-пользователя, что обеспечивает изоляцию данных при использовании приложения несколькими пользователями на одном устройстве.

## 10. Развёртывание и серверная инфраструктура

### 10.1. Платформа развёртывания

Система развёрнута на платформе Vercel, которая обеспечивает:

- **Статический хостинг** — раздача собранного SPA-приложения (каталог `dist/`) через глобальную сеть доставки контента (CDN) с автоматическим HTTPS.

- **Serverless Functions** — исполнение обработчика вебхука Telegram-бота (`api/webhook.js`) в виде бессерверной функции, масштабируемой автоматически.

- **Vercel KV** — управляемое Redis-хранилище для данных подписок.

- **Vercel Cron** — планировщик задач для ежедневной рассылки.

### 10.2. Конфигурация окружения

Секретные параметры (токен бота, URL приложения) хранятся исключительно в переменных окружения Vercel (Environment Variables) и не включаются в клиентский код. Это обеспечивает безопасность: токен бота недоступен из фронтенда и не попадает в публичный репозиторий.

### 10.3. Процесс сборки

Сборка осуществляется инструментом Vite. В процессе выполняются: транспиляция TypeScript, разрешение модулей, разделение кода на чанки (code splitting), минификация, генерация Service Worker и манифеста PWA. Результатом является каталог `dist/` со статическими файлами, готовыми к развёртыванию.

## 11. Обеспечение качества

### 11.1. Статическая типизация

Весь клиентский код написан на TypeScript со строгой конфигурацией компилятора. Интерфейсы `Card`, `Module`, `CardProgress`, `ModuleStats` и типовые псевдонимы `ReviewQuality`, `LearningMode`, `LanguageId`, `ThemeId` обеспечивают типобезопасность на всех уровнях приложения — от структуры данных до параметров функций контекста.

### 11.2. Обработка ошибок

Компонент `ErrorBoundary` перехватывает необработанные исключения в дереве React-компонентов и отображает пользователю корректное сообщение об ошибке вместо «белого экрана». Все операции с `localStorage` обёрнуты в блоки `try-catch` для обработки случаев недоступности хранилища (приватный режим браузера, исчерпание квоты).

### 11.3. Модульное тестирование

Реализованы модульные тесты для критических модулей системы с использованием фреймворка Vitest:

- `spacedRepetition.test.ts` — тестирование корректности вычислений алгоритма SM-2: инициализация прогресса, пересчёт параметров при различных оценках качества, определение карточек на повторение.
- `storage.test.ts` — тестирование функций сериализации/десериализации данных в `localStorage`.

### 11.4. Доступность (Accessibility)

Приложение соблюдает рекомендации WCAG по обеспечению доступности:

- Семантическая HTML-разметка с использованием элементов `<main>`, `<nav>`, `<section>`, `<button>`.
- Атрибуты `aria-label` для интерактивных элементов без текстового содержания.
- Стили `:focus-visible` для визуальной индикации фокуса при навигации с клавиатуры.
- Минимальный размер области нажатия (touch target) составляет 48×48 пикселей в соответствии с рекомендациями для мобильных интерфейсов.
- Атрибут `lang="ru"` на корневом элементе HTML для корректной работы программ экранного чтения.

### 11.5. Адаптивный дизайн

Интерфейс реализован по принципу «mobile-first» и корректно отображается на устройствах с различной шириной экрана. Гибкая сетка на основе CSS Flexbox обеспечивает адаптивное расположение элементов без жёстких медиа-запросов.

Программный комплекс «МГИМО BASIC ENGLISH» представляет собой комплексное решение для изучения базовой лексики английского языка, объединяющее современные подходы к построению веб-приложений (React, TypeScript, PWA), научно обоснованные методы запоминания (алгоритм интервального повторения SM-2), элементы геймификации (дневные цели, серии, XP) и удобство платформы Telegram (Mini App, чат-бот, ежедневная рассылка).
